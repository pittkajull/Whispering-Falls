# Git Rules

## Mandatory Git Workflow

* Push to GitHub after every 3 file modifications.
* Never accumulate large uncommitted changes.
* Always keep the remote repository up to date.

## Commit Requirements

* Never use co-author attribution in commits.
* Never add "Co-authored-by" trailers.
* Write commit messages in English only.
* Commit messages must be short, clear, and descriptive.

### Good Examples

* fix login validation bug
* improve dashboard loading state
* add stock prediction chart
* refactor api service layer
* update portfolio project section

### Bad Examples

* update
* fix
* changes
* final fix
* wip

## Before Every Push

* Ensure the project builds successfully.
* Verify no obvious errors are introduced.
* Keep commits focused on a single logical change whenever possible.

## Enforcement

These Git rules are mandatory and must be followed throughout the entire development process without exception.
